### Step 2 · 「按需激活」机制说明

全局安装后，Skill 不会常驻内存，而是**按需激活**。

> **重要概念**：全局安装 ≠ 常驻激活
>
> - 全局安装意味着：Skill 的配置文件永久存储在本地
> - 按需激活意味着：只有当用户输入触发词时，Skill 才加载并生效
> - 这是 AI Skill 的设计原则，避免所有Skill同时占用上下文

**激活流程：**
```
用户输入触发词
    ↓
系统检测到触发词匹配
    ↓
加载对应 Skill 的 SKILL.md 内容
    ↓
AI 按照 SKILL.md 中的角色定义响应
    ↓
Skill 进入「活跃状态」，直到当前对话结束
```

## 按需激活 vs 常驻激活 — 深度说明

**为什么 Skill 要「按需激活」而非「常驻」？**

| 维度 | 按需激活 | 常驻激活 |
|---|---|---|
| 上下文占用 | 触发后才加载，不占无关对话的上下文 | 每次对话都占用大量上下文 |
| 响应速度 | 触发时有短暂加载，但整体更快 | 响应快，但影响其他功能 |
| 多Skill共存 | 可同时安装10+个Skill，按需调用 | 同时激活>3个Skill会导致冲突 |
| 用户控制 | 用户有明确的激活/退出控制 | 用户难以明确知道哪个Skill在运行 |

**实践建议**：安装多个Skill后，明确使用触发词来激活你需要的特定Skill，避免混淆。

## 防错护栏 | Error Guards

| 错误场景 | 护栏动作 |
|---|---|
| npx 安装失败（网络错误） | 提供方案C（Python离线安装）作为备选 |
| 触发词测试只有4/7通过 | 检查未生效的触发词，修复SKILL.md的description中的触发短语格式 |
| 安装后激活响应不符合预期 | 检查 SKILL.md 的激活输出模板是否完整 |
| 仓库被删除导致安装失败 | 提示用户检查仓库状态，或使用已下载的本地文件（方案D） |
| 发现P0缺陷（安装后才发现） | 立即返回阶段05执行修复，升级patch版本 |
