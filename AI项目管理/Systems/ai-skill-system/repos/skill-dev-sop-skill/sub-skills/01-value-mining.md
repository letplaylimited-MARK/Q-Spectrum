# Sub-Skill 01 · 价值深挖 Value Mining
## Skill Dev SOP — 阶段01 · v1.0.0

---

## 职责声明 | Responsibility

本子Skill负责对用户提供的任意原始材料进行**全维度价值评估**，产出「价值地图 + 可行性评估报告」，为后续阶段提供清晰的战略方向基础。

**输入**：任意原始材料（文档/想法/代码/图片描述/口述需求/现有Skill草稿）
**输出**：结构化价值地图 + 5维可行性评估 + 推荐工程化方向（1~3个候选）
**门控**：用户确认开发方向后，方可进入阶段02

---

## 执行逻辑 | Execution Logic

### Step 1 · 材料接收与分类

接收用户材料后，首先判断材料类型：

| 材料类型 | 判断标准 | 预处理方式 |
|---|---|---|
| 文档型 | Markdown/PDF/Word/TXT | 全文阅读，提取核心概念与流程 |
| 想法型 | 口述/笔记/关键词 | 追问5W1H，补全信息 |
| 代码型 | 脚本/函数/API | 分析功能逻辑，识别可Skill化部分 |
| 混合型 | 多材料组合 | 逐一分类后整合 |
| 现有Skill草稿 | 已有SKILL.md或提示词 | 进入「优化模式」，差距分析 |

**若材料不完整**，在此阶段追问，绝不带着模糊假设进入下一阶段：

> 「我注意到你的材料中关于[X]部分还不够清晰，请告诉我：  
> 1. [具体缺失信息A]  
> 2. [具体缺失信息B]  
> 收到你的补充后，我将开始完整的价值分析。」

---

### Step 2 · 五维价值地图生成

对原始材料从以下5个维度输出分析：

**维度1 · 核心价值（Core Value）**

- 这个材料/想法解决什么**本质问题**？
- 现有解决方案（人工/其他工具/其他Skill）的痛点是什么？
- 如果做成Skill，用户能省去什么操作？节省多少时间/精力？

**维度2 · 场景矩阵（Scenario Matrix）**

| 用户类型 | 使用场景 | 使用频率 | 价值强度(1-5) |
|---|---|---|---|
| 个人用户 | [场景描述] | 低/中/高 | ⭐⭐⭐ |
| 开发者 | [场景描述] | 低/中/高 | ⭐⭐⭐⭐ |
| 企业用户 | [场景描述] | 低/中/高 | ⭐⭐⭐⭐⭐ |

**维度3 · 工程化潜力（Engineering Potential）**

对材料中每个核心模块评估Skill化适合程度（1-10分）：

| 模块名称 | Skill化评分 | 评分理由 | 优先级 |
|---|---|---|---|
| [模块A] | 9/10 | 高频/通用/独立性强 | P0 |
| [模块B] | 7/10 | 有价值但依赖上下文 | P1 |
| [模块C] | 4/10 | 过于专业化，受众窄 | P2 |

**维度4 · 局限性缺口（Gap Analysis）**

识别原始材料的3个最大不足（做成Skill后需要补足的部分）：

1. **[缺口1]**：[描述] → Skill化补偿方案：[方案]
2. **[缺口2]**：[描述] → Skill化补偿方案：[方案]
3. **[缺口3]**：[描述] → Skill化补偿方案：[方案]

**维度5 · 差异化优势（Differentiation）**

- 相比现有同类Skill/工具，本材料独特在哪里？
- 技术护城河：什么是难以被轻易复制的？
- 目标用户最在意的1~2个核心差异点

### Step 4 · 推荐开发方向（候选菜单）

基于价值地图，提供1~3个方向供用户选择：

```
📍 推荐开发方向

方向A（最小可行版本 MVP）
  → Skill名称：[候选名]
  → 核心功能：[1句话描述]
  → 预计完成：阶段01~07，约[X]轮对话
  → 优势：快速上线，验证核心价值

方向B（完整增强版）
  → Skill名称：[候选名]
  → 核心功能：[1句话描述]
  → 预计完成：全10阶段，约[X]轮对话
  → 优势：功能完整，用户体验优秀

方向C（专项模块版）  ← 如有必要
  → Skill名称：[候选名]
  → 核心功能：[1句话描述]
  → 预计完成：阶段01~07，约[X]轮对话
  → 优势：聚焦单一场景，精准高效
```

**门控提问：**

> 「以上是我对你的材料的完整价值分析。请选择开发方向（A/B/C），或告诉我你的想法——我将据此生成设计约束书，正式启动工程化阶段。」

## 防错护栏 | Error Guards

| 错误场景 | 护栏动作 |
|---|---|
| 材料为空/极度模糊 | 输出追问清单，不进行价值分析 |
| 用户跳过直接要求「开始做」 | 提示：「价值分析是工程质量的地基，请先确认方向再构建，避免返工」 |
| 材料超长（>5000字） | 先提取摘要，请用户确认核心部分后再全面分析 |
| 材料包含多个独立想法 | 分别列出，建议选择1个优先开发，其余记录为「未来版本候选」 |
| 用户要求「全部都做」 | 建议MVP方式：先完成一个，其余进入路线图 |
