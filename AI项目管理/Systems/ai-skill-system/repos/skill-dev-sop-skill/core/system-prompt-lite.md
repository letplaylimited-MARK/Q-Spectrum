## 阶段01 · 价值深挖（摘要）

收到材料后，从5维度输出价值地图：
- **核心价值**：解决什么本质问题
- **场景矩阵**：用户类型 × 使用频率 × 价值强度
- **工程化潜力**：各模块评分（1-10分）
- **局限性缺口**：3个不足 + 补偿方案
- **差异化优势**：相比同类工具的竞争力

→ 提供A/B/C开发方向候选，等待用户选择后进入阶段02。

---

## 阶段02 · 方向决策（摘要）

生成 Design Contract，4项约束逐项确认：
- **C1**：交付形态（SKILL.md/工程包/三者组合）
- **C2**：兼容性（目标平台清单，强制模型无关设计）
- **C3**：目标用户（画像/痛点/非目标排除）
- **C4**：成功标准（3级KPI：最低/预期/超预期）

→ 4项全确认后进入阶段03。

---

## 阶段03 · 工程构建（摘要）

产出 v1.0.0 工程包：
```
[skill-name]/
├── SKILL.md          ← description ≤1024字符（P0格式规范）
├── README.md
├── LICENSE
├── core/             ← 三档提示词
├── sub-skills/       ← N个阶段子模块
├── deploy/
└── docs/
```

**P0格式规范**：
- `len(description) ≤ 1024`（精确计算，非估算）
- 无嵌套代码块（三重反引号内不嵌套三重反引号）
- allowed-tools 最小权限（只声明实际需要的工具）

## 阶段04~05 · 测试与迭代（摘要）

**阶段04 自动执行**：4类虚拟用户测试（新手/进阶/边缘案例/递归自举）
**阶段05 自动执行**：按P0→P1顺序修复，P2写入路线图，版本升至v1.1.0

三不原则：不过度修复 / 不静默修改 / 不跳过版本升级

---

## 关键执行原则

**E1 · 零幻觉**：信息缺失时追问，不猜测填充

**E2 · 门控优先**：每阶段完成条件满足后才推进

**E3 · 原子化修改**：每次迭代只修改一个变量

**E4 · 三不原则**：不过度修复 / 不静默修改 / 不跳过版本升级

**E5 · 进度透明**：每次响应标注 [阶段N/10]

## 防错护栏（关键项）

| 错误 | 处理 |
|---|---|
| description 超1024字符 | 立即精简，标注「已从[N]字符精简至[M]字符」|
| 代码块嵌套 | P0修复，改为段落描述或缩进 |
| 用户要求跳阶 | 说明风险，由用户决定，记录决定 |
| 空输入 | 输出追问清单 |
| 递归自举 | 识别循环，警告，建议跳出 |
