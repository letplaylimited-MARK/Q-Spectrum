# Sub-Skill 01 · 验收材料接收器

**所属 Skill**：Skill 05 · Validator  
**版本**：v1.0.0  
**职责**：接收、验证并解析来自上游 Skill 的交接包和工程材料

---

## 支持的输入来源

| 来源 | 期望材料 | 交接包类型 |
|------|---------|-----------|
| Skill 04 (Planner) | 操作手册 + 交接包 | s04-to-user.yaml 扩展格式 |
| Skill 02 (SOP 工程师) | SOP 文档 + 交接包 | s02-to-s05.yaml |
| 用户直接提交 | 任意格式材料 | 无标准交接包 |

---

## 交接包接收与验证

### Schema 验证规则

```yaml
# 必须字段检查
required_fields:
  - schema_version     # 必须为 "1.0"
  - from_skill         # 来源 Skill 标识
  - to_skill           # 必须包含 "skill-05"
  - payload            # 必须非空

# 可选但重要的字段
optional_important:
  - test_targets       # 验收标准列表
  - quality_requirements  # 质量指标
  - defect_history     # 历史缺陷（有则优先复验）
```

### 接收确认输出

```
📥 验收材料接收确认

来源：{from_skill}
项目：{project_name} v{version}
材料类型：{操作手册/SOP文档/混合}

识别到的验收标准：
{N} 个测试目标
{M} 项质量要求

历史缺陷记录：{有 X 条 / 无}

接下来将按以下顺序验收：
1. 功能完整性（40%）
2. 文档完整性（25%）
3. 可执行性（20%）
4. 接口规范性（10%）
5. 安全性（5%）

预计生成报告时间：{短/中/长}（根据材料复杂度）
```

## 无交接包时的降级处理

当用户直接提交材料（无标准交接包）时：

```
未检测到标准交接包格式。

将对您提交的材料执行通用验收测试。

请补充以下信息：
1. 项目名称和版本
2. 主要功能是什么（用于验收功能完整性）
3. 您认为什么是"验收通过"的标准

或直接说"使用默认验收标准"，我将按通用AI项目标准执行验收。
```

## 历史缺陷优先复验规则

当 `defect_history` 非空时：

```yaml
priority_recheck:
  rule: "历史 P0/P1 缺陷在新版本中优先复验"
  order:
    1: "P0 历史缺陷 → 必须首先验证已修复"
    2: "P1 历史缺陷 → 第二优先级"
    3: "新增测试项 → 最后执行"
  
  report_format: |
    ## 历史缺陷复验结果
    | 缺陷 ID | 原缺陷描述 | 本次状态 | 说明 |
    |---------|-----------|---------|------|
    | {ID} | {描述} | ✅已修复/❌仍存在/⚠️部分修复 | {备注} |
```
