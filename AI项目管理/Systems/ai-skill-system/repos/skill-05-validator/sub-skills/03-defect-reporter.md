# Sub-Skill 03 · 缺陷分类与报告生成器

**所属 Skill**：Skill 05 · Validator  
**版本**：v1.0.0  
**职责**：将验收发现的问题分类为 P0-P3 缺陷，生成标准化验收报告

---

## 缺陷优先级定义

| 级别 | 名称 | 定义 | 发布影响 |
|------|------|------|---------|
| **P0** | 阻塞级 | 安全漏洞/核心功能失效/数据丢失风险/接口版本不兼容 | 立即 FAIL，不可协商 |
| **P1** | 严重级 | 功能不完整/关键步骤缺失/验证方式大量缺失/未使用环境变量管理凭证 | P1>3 则 FAIL |
| **P2** | 一般级 | 文档不清晰/边界条件未处理/断点恢复缺失/FAQ 不足 | 建议修复，不阻塞 |
| **P3** | 轻微级 | 格式不统一/命名不规范/可优化的表达 | 记录，不要求修复 |

---

## 缺陷记录格式

```yaml
defect:
  id: "{LEVEL}-{序号}"           # 如 P1-001
  level: "P0/P1/P2/P3"
  dimension: "functional/documentation/executability/interface/security"
  description: "{缺陷描述（一句话）}"
  location: "{文件名/章节/步骤ID}"
  evidence: |
    {可重现的证据或引用原文}
  impact: "{对用户的实际影响}"
  fix_suggestion: |
    {具体修复建议}
  fix_route: "skill-0X-name"      # 应路由到哪个 Skill 修复
  status: "open"                   # open/fixed/wont_fix/deferred
```

---

## 修复路由规则

| 缺陷类型 | 路由到 | 说明 |
|---------|-------|------|
| 功能不完整 | Skill 02 | SOP 工程师重新设计 |
| 技术选型问题 | Skill 03 | 开源侦察官重新评估 |
| 需求理解偏差 | Skill 01 | 需求翻译官重新分析 |
| 步骤不可执行 | Skill 04 | 执行规划官修订手册 |
| 幻觉防护缺失 | 对应生成 Skill | 在模板中增加标注 |
| 接口格式错误 | 来源 Skill | 修正交接包格式 |

---

## 发布决策矩阵

```
P0 缺陷数量
    │
    ├── ≥1 → FAIL（无论其他指标）
    │
    └── =0 ─────────────────────────────────────────
                                                    │
                        P1 缺陷数量                 │
                            │                       │
                    ┌───────┴───────┐               │
                   =0             1-3           >3  │
                    │               │               │
              综合评分         综合评分          FAIL │
                  │                │
             ≥80% | <80%      ≥70% | <70%
              │       │        │       │
            PASS  COND_PASS  COND_PASS FAIL
```

**CONDITIONAL_PASS 条件**：必须注明 P1 缺陷修复期限（建议 3-7 个工作日）
