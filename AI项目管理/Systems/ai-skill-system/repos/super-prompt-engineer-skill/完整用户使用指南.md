## 第一章：你是哪类用户？（2分钟自我定位）

在开始之前，先花 2 分钟确认你属于哪类用户——不同类型对应不同的推荐路径，**选对路径可以节省 80% 的摸索时间**。

### 1.1 用户类型速判表

| 描述自己 | 用户类型 | 推荐路径 |
|---|---|---|
| 我不懂技术，只是想让 AI 帮我更好地完成日常工作 | **🟢 零基础用户** | 场景 A 或 B → 第三章 → 第四章 |
| 我是内容创作者、自媒体、写手，每天大量使用 AI | **🔵 创作型用户** | 场景 C 或 E → 第三章 → 第四章 |
| 我管理一个团队，希望统一 AI 使用规范 | **🟡 企业管理者** | 场景 D 或 F → 场景 K → 第五章 |
| 我会写代码，想把这个 Skill 接入自己的产品/系统 | **🔴 开发者** | 场景 J → 第五章 → 第六章 |
| 我在用 CodeBuddy，想直接安装 Skill | **⚡ CodeBuddy 用户** | 场景 I → 第三章 |

> **提示**：如果你同时符合多个类型，以「最主要的使用场景」为准。

### 1.2 版本选择指引

在开始部署前，先确认用哪个版本的系统提示词：

| 你的情况 | 推荐版本 | 对应文件 |
|---|---|---|
| 使用 GPT-4o / Claude 3 / Qwen3.5 等主流强模型 | **完整版** | `core/system-prompt-full.md` |
| 使用免费版 / 移动端 / 轻量模型（上下文 ≤ 4K） | **精简版** | `core/system-prompt-lite.md` |
| API 调用，需要控制 Token 成本 | **精简版** | `core/system-prompt-lite.md` |
| 只需要某一项能力（如只优化提示词） | **单项 Sub-Skill** | `sub-skills/0X-xxx.md` |

## 第二章：各场景完整部署方式

<a name="场景A"></a>
### 场景 A：网页对话直接粘贴（最简单 · 5分钟 · 零门槛）

**适合人群**：所有用户，尤其是第一次使用、想快速体验的用户  
**优点**：无需任何账号或配置，5分钟内可运行  
**缺点**：仅在当前对话生效，关闭后需重新激活  
**支持平台**：ChatGPT / Claude / 文心一言 / 通义千问 / 讯飞星火 / Kimi / 豆包 / 任意网页版 AI

#### 操作步骤

**Step 1**：打开 `core/system-prompt-full.md` 文件，**全选并复制**全部内容。

**Step 2**：打开你常用的 AI 对话窗口，新建一个对话。

**Step 3**：将复制的内容**直接粘贴到对话输入框**，发送。

**Step 4**：等待 AI 输出激活确认信息（见第三章验证方法）。

**Step 5**：激活成功后，在**同一个对话窗口**内正常使用即可。

#### 注意事项

> ⚠️ **重要**：粘贴的是「系统提示词」，不是给 AI 提问。粘贴后直接发送，不需要加任何前缀（如「请你…」「你现在是…」）。

> ⚠️ **会话绑定**：Skill 仅在当前对话生效。如需持久化，请使用场景 C（ChatGPT）或场景 E（Claude）。

---

<a name="场景B"></a>
### 场景 B：文件上传投喂（最直接 · 3分钟）

**适合人群**：支持文件上传的平台用户  
**优点**：操作极简，无需复制长文本  
**支持平台**：Claude（支持上传 .md 文件）/ ChatGPT Plus（支持附件）/ Kimi（支持文件上传）

#### 操作步骤

**Step 1**：打开新对话。

**Step 2**：点击对话框的「附件/上传」按钮，上传 `core/system-prompt-full.md` 文件。

**Step 3**：在文件上传后，发送以下消息触发激活：

```
请完整阅读我上传的文件，并按照文件中的指示激活对应角色与能力。
```

**Step 4**：等待激活确认（见第三章）。

#### Claude 用户额外说明

Claude 对上传文件的读取非常可靠，推荐此方式。上传后无需额外指令，AI 会自动识别系统提示词格式并执行激活。

<a name="场景C"></a>
### 场景 C：ChatGPT Custom Instructions（持久化 · 个人推荐）

**适合人群**：ChatGPT 注册用户，希望每次对话自动激活，无需重复设置  
**优点**：一次设置，永久生效；每个新对话自动加载  
**缺点**：字符数有上限（约 1500 字），建议使用精简版  
**需要**：ChatGPT 账号（免费版或 Plus 均可）

#### 操作步骤

**Step 1**：登录 ChatGPT，点击右上角头像 → **Settings（设置）**。

**Step 2**：选择 **Personalization（个性化）** → **Custom Instructions（自定义指令）**。

**Step 3**：页面有两个文本框：
- 上方框：「What would you like ChatGPT to know about you?」→ **留空或填写你的个人信息**
- 下方框：「How would you like ChatGPT to respond?」→ **粘贴 `core/system-prompt-lite.md` 全文**

**Step 4**：点击 **Save（保存）**。

**Step 5**：新建一个对话，验证激活状态（见第三章）。

#### 使用体验说明

设置完成后，每次打开新对话，AI 都会在后台加载这份 Skill，你无需每次手动激活。直接对话即可使用所有 6 项能力。

<a name="场景D"></a>
### 场景 D：ChatGPT GPTs 定制机器人（团队共享 · 可分发）

**适合人群**：希望将 Skill 打包成一个机器人分享给团队或公开发布的用户  
**优点**：生成专属链接，团队成员点击即用；可设置开场白和快捷按钮  
**需要**：ChatGPT Plus 账号

#### 操作步骤

**Step 1**：登录 ChatGPT → 左侧菜单 → **Explore GPTs** → 右上角 **+ Create**。

**Step 2**：进入 GPT 编辑页面，选择 **Configure（配置）** 标签页。

**Step 3**：填写以下信息：

| 字段 | 填写内容 |
|---|---|
| Name 名称 | 超级提示词工程师 / Super Prompt Engineer |
| Description 描述 | 通用AI提示词优化、工作流设计、报告生成的专业助手 |
| Instructions 指令 | 粘贴 `core/system-prompt-full.md` 全文 |
| Conversation starters | 帮我优化这个提示词 / 设计一个工作流 / 梳理我的使用痛点 |

**Step 4**：点击右上角 **Save（保存）**，选择分享范围：
- **Only me**：仅自己使用
- **Anyone with the link**：生成链接，分享给团队
- **Public**：公开发布到 GPT 商店

**Step 5**：复制链接，发送给团队成员，他们点击链接即可直接使用，无需任何配置。

<a name="场景E"></a>
### 场景 E：Claude Projects（强力推荐 · 最佳体验）

**适合人群**：Claude 用户，尤其是需要长期使用、需要上传文档辅助工作的用户  
**优点**：Projects 功能支持持久系统提示词 + 知识库上传，是目前最接近「完整 Skill 体验」的方案  
**需要**：Claude.ai 账号（Pro 版本支持 Projects）

#### 操作步骤

**Step 1**：登录 [claude.ai](https://claude.ai) → 左侧 **Projects** → **+ New Project**。

**Step 2**：进入项目后，点击右上角 **Project Instructions（项目指令）**。

**Step 3**：将 `core/system-prompt-full.md` 全文粘贴至指令框，保存。

**Step 4**（可选，强烈推荐）：点击 **+ Add content（添加内容）**，上传以下文件作为知识库：
- `sub-skills/01-prompt-optimizer.md`
- `sub-skills/02-workflow-designer.md`
- `sub-skills/03-report-generator.md`
- 其他你常用的子 Skill 文件

**Step 5**：在该 Project 下新建对话，AI 自动加载 Skill + 知识库，立即可用。

#### 为什么 Claude Projects 是最佳选择？

Claude Projects 的优势在于：
1. **持久系统提示词**：项目内所有对话共享同一套 Skill 配置，无需重复激活
2. **知识库集成**：上传的子 Skill 文件可被 Claude 主动检索，实现真正的模板按需调用
3. **对话历史共享**：同一 Project 内的多次对话可相互参考，部分实现跨会话记忆

<a name="场景F"></a>
### 场景 F：Coze Bot 部署（字节跳动 · 中国平台推荐）

**适合人群**：希望在豆包/Coze 平台部署，或需要接入微信、飞书等国内渠道  
**优点**：免费；支持接入微信公众号、飞书机器人等企业级渠道；支持可视化工作流  
**网址**：[www.coze.cn](https://www.coze.cn)（国内）/ [www.coze.com](https://www.coze.com)（国际）

#### 基础部署步骤

**Step 1**：注册登录 Coze → **创建 Bot**。

**Step 2**：在 Bot 配置页面：
- **人设与回复逻辑**：粘贴 `core/system-prompt-full.md` 全文
- **开场白**：填写「超级提示词工程师已就绪，请告诉我你想解决什么问题？」
- **快捷指令**：添加「优化提示词 / 设计工作流 / 生成报告 / 梳理痛点」

**Step 3**：选择底层模型（推荐：豆包 Pro / GPT-4o / Claude 3.5）。

**Step 4**：点击**发布**，选择发布渠道：
- 个人使用：Coze 内部链接
- 团队使用：飞书机器人 / 企业微信
- 公开发布：Coze 机器人商店

#### 进阶：可视化工作流配置

Coze 支持将 6 个子 Skill 映射为工作流节点，实现完整的 Skill 流水线：

```
[用户输入] → [痛点引导节点] → [提示词优化节点] → [工作流设计节点] → [报告生成节点] → [输出]
```

进入 Bot 配置 → **工作流** → 参考 `sub-skills/` 目录中各文件的执行步骤设计节点。

<a name="场景J"></a>
### 场景 J：API 调用（开发者 · 完整集成）

**适合人群**：需要将 Skill 集成到自有产品、自动化流程或代码工作流中的开发者  
**支持**：OpenAI API / Anthropic API / 阿里云 DashScope / 百度千帆 / 讯飞 WebSocket API

#### Python 完整示例

```python
# ===== super_prompt_engineer.py =====
# 完整的 Super Prompt Engineer Skill API 调用示例

import openai
from pathlib import Path

class SuperPromptEngineer:
    """超级提示词工程师 Skill 的 Python 封装"""
    
    def __init__(self, model="gpt-4o", api_key=None, use_lite=False):
        """
        初始化
        :param model: 使用的模型名称
        :param api_key: API 密钥（也可通过环境变量 OPENAI_API_KEY 设置）
        :param use_lite: 是否使用精简版（Token 受限时使用）
        """
        self.client = openai.OpenAI(api_key=api_key)
        self.model = model
        
        # 加载系统提示词
        skill_dir = Path(__file__).parent  # 指向 skill 目录
        prompt_file = "system-prompt-lite.md" if use_lite else "system-prompt-full.md"
        self.system_prompt = (skill_dir / "core" / prompt_file).read_text(encoding="utf-8")
        
        # 对话历史（支持多轮对话）
        self.history = []
    
    def chat(self, user_message: str) -> str:
        """
        发送消息并获取响应
        :param user_message: 用户输入
        :return: AI 响应文本
        """
        self.history.append({"role": "user", "content": user_message})
        
        response = self.client.chat.completions.create(
            model=self.model,
            messages=[
                {"role": "system", "content": self.system_prompt},
                *self.history
            ],
            temperature=0  # 执行类任务保持确定性
        )
        
        assistant_message = response.choices[0].message.content
        self.history.append({"role": "assistant", "content": assistant_message})
        return assistant_message
    
    def reset(self):
        """重置对话历史"""
        self.history = []


# ===== 使用示例 =====
if __name__ == "__main__":
    # 初始化（确保环境变量 OPENAI_API_KEY 已设置）
    engineer = SuperPromptEngineer(model="gpt-4o")
    
    # 示例 1：提示词优化
    print("=== 提示词优化示例 ===")
    result = engineer.chat("优化这个提示词：帮我写小红书")
    print(result)
    
    # 示例 2：工作流设计
    engineer.reset()  # 新任务前重置对话
    print("\n=== 工作流设计示例 ===")
    result = engineer.chat("帮我设计一个每日内容创作的工作流，使用 AI 辅助，每天产出 3 篇文章")
    print(result)
```

#### Node.js / TypeScript 示例

```typescript
// ===== superPromptEngineer.ts =====
import OpenAI from 'openai';
import { readFileSync } from 'fs';
import { join } from 'path';

const client = new OpenAI(); // 从环境变量读取 OPENAI_API_KEY

// 加载系统提示词
const systemPrompt = readFileSync(
  join(__dirname, 'core', 'system-prompt-full.md'),
  'utf-8'
);

// 对话历史（多轮对话支持）
const conversationHistory: OpenAI.ChatCompletionMessageParam[] = [];

async function chat(userMessage: string): Promise<string> {
  conversationHistory.push({ role: 'user', content: userMessage });
  
  const response = await client.chat.completions.create({
    model: 'gpt-4o',
    messages: [
      { role: 'system', content: systemPrompt },
      ...conversationHistory
    ],
    temperature: 0
  });
  
  const reply = response.choices[0].message.content ?? '';
  conversationHistory.push({ role: 'assistant', content: reply });
  return reply;
}

// 使用示例
(async () => {
  const result = await chat('帮我优化这个提示词：帮我写文章');
  console.log(result);
})();
```

### 3.2 激活失败的处理方案

| 现象 | 原因 | 解决方案 |
|---|---|---|
| AI 没有输出激活确认，正常聊天 | 内容未被识别为系统提示词 | 在消息开头加「请阅读以下系统配置并激活对应角色：」再粘贴 |

<a name="第六章"></a>
## 第六章：常见问题与排错

### Q1：激活后 AI 的回答格式乱了，不按模板输出

**原因**：模型理解能力不足，或上下文过长导致指令遗忘

**解决方案**：
1. 尝试在每次请求前加一句「请按照标准输出格式回复」
2. 改用更强的模型（推荐 GPT-4o 或 Claude 3.5）
3. 若上下文过长，新建对话重新激活

---

### Q2：提示词优化后在某个模型效果变差了

**原因**：不同模型对提示词的解析偏好不同

**解决方案**：
1. 告诉 AI「这个提示词专门用于[模型名称]，请针对该模型的特点优化」
2. 如果需要兼容多模型，可以请 AI 生成「多模型版本」

---

### Q3：痛点引导的问题太多，我不知道怎么回答

**解决方案**：
1. 不需要每个问题都详细回答，描述「最困扰你的 1-2 个场景」即可
2. 可以说「我不知道怎么描述，你帮我举几个例子」，AI 会提供参考选项

---

### Q4：生成的工作流步骤太细了，有些步骤我不需要

**解决方案**：

```
这个工作流步骤太细了，帮我合并/简化，保留核心的 3-5 个步骤
```

---

### Q5：我想只用某一项能力，不想加载全部 Skill

**解决方案**：使用对应的单项 Sub-Skill 文件，例如只需要提示词优化，就只投喂 `sub-skills/01-prompt-optimizer.md`，文件更小，加载更快，效果更专注。

---

### Q6：在中文模型上使用，输出质量不如英文模型

**原因**：系统提示词为中英双语，中文模型有时会偏向英文部分

**解决方案**：在激活后添加一句：

```
请全程使用中文输出，按照中文版指令执行所有任务。
```

---

### Q7：API 调用时每次都要发送很长的系统提示词，成本很高

**解决方案**：
1. 改用 `system-prompt-lite.md` 精简版（约 300 词 vs 完整版约 1000 词）
2. 将系统提示词缓存，避免重复计费（OpenAI 的 Prompt Caching 功能）
3. 只加载需要的单项 Sub-Skill 文件
