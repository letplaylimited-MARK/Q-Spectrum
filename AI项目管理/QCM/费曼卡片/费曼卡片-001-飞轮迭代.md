## 🎯 核心概念

**飞轮迭代（Flywheel Iteration）**

飞轮迭代是一种自我进化的执行框架，模拟物理飞轮的动量积累效应：
- 每一次迭代都在上一次基础上增加动量
- 不会停下来，只会越转越快
- 小幅度的持续改进 > 大幅度的间歇性重构

---

## 📚 通俗解释

### 比喻：骑自行车

想象你骑自行车启动的那一刻：
1. **静止时**：需要很大力气才能开始动
2. **启动后**：只需要持续踩踏，速度越来越快
3. **高速时**：微小的调整就能保持平衡

飞轮迭代就是这样——前期建立系统费力气，但一旦运转起来，每次迭代都在积累势能。

### 公式表达

```
动量 = Σ(每次迭代投入 × 效率系数)
效率系数 = f(发现问题能力, 执行能力, 反思深度)
```

---

## 🔬 技术细节

### 6阶段循环

```
Scan (扫描) → Diagnose (诊断) → Plan (规划) 
      ↑                                    ↓
Crystallize ← Validate ← Execute (执行)
```

| 阶段 | 输入 | 输出 | 关键指标 |
|------|------|------|----------|
| Scan | 工作区状态 | 问题列表 | 文件数/目录数 |
| Diagnose | 问题 | 诊断报告 | P0/P1/P2分类 |
| Plan | 诊断 | 执行计划 | 任务数/优先级 |
| Execute | 计划 | 执行结果 | 成功率 |
| Validate | 结果 | 验证报告 | 验证分数 |
| Crystallize | 验证 | 结晶 | 新知识/经验 |

### 评分机制

```
综合评分 = 0.4×阶段完成率 + 0.3×任务成功率 + 0.3×验证分数
```

## 💡 应用场景

- **日常自检**：每天运行飞轮扫描发现问题
- **项目迭代**：按周运行完整6阶段
- **版本规划**：按月进行战略规划
- **持续优化**：每轮迭代后更新MEMORY.md
