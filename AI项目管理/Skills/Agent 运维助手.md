# Skill 09 · Agent 运维助手 (v2.0 Final)

## 激活词
Agent 运维 / 自动化运维 / 帮我执行 [任务名] / 生成本周周报 / 全库盘点 / 检查记忆完整性

## 使用时机
当遇到以下情况时触发此 Skill：
1. 用户不想安装或运行 Python 环境。
2. 用户希望通过简单的对话指令完成复杂的系统检查或文件操作。
3. 需要将现有的脚本逻辑转化为 AI 原生能力。

## 核心工作流

### 任务 A：记忆系统完整性检查
1. **扫描目录**: 读取 `Systems/operator/` 及其子目录。
2. **核对清单**: 检查必需文件是否存在且非空。
3. **输出报告**: 生成状态表格。

### 任务 B：周报自动生成
1. **扫描变动**: 识别最近新增或修改的文件。
2. **提取关键信息**: 从文件名提取核心变动。
3. **生成草稿**: 生成 `Week-XX.md` 草稿。
4. **交付确认**: 展示草稿等待审核。

### 任务 C：全库盘点与健康度评估
1. **文件统计**: 统计核心目录文件总数。
2. **三表一致性检查**: 对比需求池、技能池、飞轮总表数据。
3. **输出报告**: 生成盘点报告。

### 任务 D：标签系统管理
1. **扫描未打标文件**: 查找缺失标签的文件。
2. **智能匹配**: 自动匹配最合适的标签。
3. **追加标签**: 添加标准化标签块。

## 交付物
执行完成后，应产出：
- 📝 **任务报告**: 包含检查结果、变动统计或生成的草稿。
- 💾 **持久化文件**: 如周报草稿或盘点报告。

## 验证方法

| 验证标准 | 通过条件 | 验证方式 |
|----------|----------|----------|
| 文件检查准确性 | 100% 识别缺失/存在文件 | 手动核对 |
| 报告完整性 | 包含所有必需章节 | 检查输出格式 |
| 执行安全性 | 无未授权删除操作 | 审计日志 |

## 常见错误提醒

| 错误 | 表现 | 正确做法 |
|------|------|----------|
| 依赖外部脚本 | 要求用户运行 Python | 使用 AI 原生文件操作 |
| 盲目删除 | 直接删除文件 | 先请示用户 |
| 路径错误 | 使用有空格路径 | 严格校验路径 |
| 幻觉生成 | 捏造统计数据 | 基于实际读取 |
