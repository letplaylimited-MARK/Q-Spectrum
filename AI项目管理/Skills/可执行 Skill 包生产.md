# Skill 03 · 可执行 Skill 包生产

## 🎯 激活词
- `生产 Skill`
- `文档转 Skill`
- `Skill 工程化`

## ⏱️ 使用时机
当你有成熟的专题文档或方法论，想将其转化为标准化、可复用的 Skill 包时触发。

## 🛠️ 核心工作流
1. **提取核心逻辑**: 从原文档中剥离背景描述，保留核心步骤、规则和交付物。
2. **结构化封装**: 按照标准模板（激活词/时机/工作流/交付物/避坑）重组内容。
3. **测试验证**: 模拟用户输入，验证 Skill 输出是否符合预期。
4. **发布入库**: 将文件存入 `Skills/` 目录，并更新 `技能机会总表.md`。

## 📦 交付物
- 📦 标准化 Skill 文档 (`.md`)
- 📝 入库记录 (更新总表状态)

## ⚠️ 常见错误提醒
- ❌ 照搬原文 -> ✅ 必须精简，只保留可执行指令。
- ❌ 缺乏测试 -> ✅ 生产后必须进行至少一次模拟调用。

## 验证方法

| 验证标准 | 通过条件 | 验证方式 |

| 可调用性 | 能被直接作为 Skill 使用 | 模拟调用 |
| 入库完整性 | 已更新总表和索引 | 检查总表 |
