## Case 1: Solo Entrepreneur — Launch an AI Startup
## 案例 1：獨立創業者 — 啟動AI創業

**Who this is for / 适用人群:**
Non-technical founders who have a product idea but need a full team to think it through.
适合有产品想法但需要完整团队来推进思考的非技术创始人。

**Prompt — copy and paste this:**

```
I want to build an AI-powered tutoring platform. Help me from idea to MVP.
```

**Roles that activate:** T02 (Strategy) → Q02 (Research) → Q01 (Architecture) → Q03 (Content) → Q04 (Analysis)

**What the AI will do:**
1. T02 frames the venture — market positioning, competitive landscape, business model canvas
2. Q02 researches existing tutoring platforms, identifies gaps, validates the niche
3. Q01 designs the technical architecture — tech stack, database schema, API structure
4. Q03 drafts onboarding copy, landing page content, and user communication
5. Q04 builds a metrics framework — KPIs, user acquisition projections, cost analysis

**Follow-up prompts to deepen the work:**

```
Now create a 90-day launch roadmap with weekly milestones.
```

```
Draft the pitch deck for seed funding — 10 slides max.
```

```
What are the top 5 risks and how do we mitigate each one?
```

## Case 9: Mental Health — Personal Support
## 案例 9：心理健康 — 個人支持

**Who this is for / 适用人群:**
Anyone experiencing stress, burnout, or emotional difficulty who wants a thoughtful conversation.
适合任何经历压力、倦怠或情绪困难并希望进行深度对话的人。

**Prompt — copy and paste this:**

```
I'm feeling burned out from work. Can we talk about it?
```

**Roles that activate:** Q07 (Emotional Support)

**What the AI will do:**
1. Q07 engages in empathetic, non-judgmental conversation
2. No task-switching or role-hopping — the AI stays in support mode
3. Listens, reflects, validates, and gently explores coping strategies
4. May suggest practical micro-steps (boundaries, rest, journaling) without being prescriptive

**Follow-up prompts to continue the conversation:**

```
What are some realistic boundaries I can set with my manager this week?
```

```
Help me design a simple evening wind-down routine — 30 minutes max.
```

```
I think part of it is imposter syndrome. Can we explore that?
```

> **Note / 备注:** This role prioritizes emotional safety. It will not pivot to productivity advice unless you explicitly ask. Q-SpecTrum understands that sometimes you need a listener, not a strategist.
> 此角色优先保障情感安全。除非您明确要求，否则不会转向生产力建议。Q-SpecTrum 理解有时您需要的是倾听者，而非策略师。

## Create Your Own / 創建自己的場景

You are not limited to the 10 cases above. Q-SpecTrum routes your request to the right roles based on what you say. Here is how to get the best results:

您不必局限于以上 10 个案例。Q-SpecTrum 会根据您的描述将请求路由到正确的角色。以下是获得最佳结果的方法：

### Tip 1: Be specific about your situation / 具体描述您的情况

| Instead of / 不要这样说 | Try this / 试试这样说 |
|---|---|
| "Help me with marketing" | "I run a B2B SaaS tool for HR teams. Help me create a LinkedIn content strategy targeting CHROs." |
| "帮我做营销" | "我运营一个面向HR团队的B2B SaaS工具。帮我创建一个针对CHRO的LinkedIn内容策略。" |

### Tip 2: State your desired outcome / 说明期望的结果

| Instead of / 不要这样说 | Try this / 试试这样说 |
|---|---|
| "Look at my code" | "Review my authentication module for security vulnerabilities and give me a prioritized fix list." |
| "看看我的代码" | "审查我的认证模块的安全漏洞，并给我一个按优先级排序的修复清单。" |

### Tip 3: Mention constraints / 提及约束条件

```
I have a $5,000 budget and 3 weeks. Help me launch a landing page
that converts at 5%+ for my SaaS product.
```

```
我有5000美元预算和3周时间。帮我启动一个转化率超过5%的SaaS产品着陆页。
```

### Tip 4: Chain your requests / 链式请求

Start broad, then drill down. Each follow-up prompt keeps the context alive and can activate new roles:

从宏观开始，然后逐步深入。每个后续提示都会保持上下文，并可能激活新角色：

```
Step 1: "Help me plan a mobile app for pet owners."        → T02, Q02
Step 2: "Now design the database schema."                  → Q01
Step 3: "Write the onboarding email sequence."             → Q03
Step 4: "What are the legal risks in storing pet health data?" → Q06
```

### Tip 5: The golden rule / 黄金法则

> **The more context you give, the better the routing.**
> You do not need to name the roles. Just describe what you need — Q-SpecTrum figures out the rest.
>
> **您提供的上下文越多，路由就越精准。**
> 您不需要指定角色名称。只需描述您的需求——Q-SpecTrum 会自动完成其余工作。
