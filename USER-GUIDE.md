# Q-SpecTrum User Guide / 使用指南

> **For**: Anyone using Q-SpecTrum with any AI model
> **Assumes**: You have already installed and activated Q-SpecTrum (see INSTALL-GUIDE.md)
> **Version**: 1.0 (2026-04-20)

## 1. How Q-SpecTrum Works / 工作原理

### The Shared Brain / 共同大腦

Q-SpecTrum is not software you install — it is a **folder that thinks with your AI**.

| Q-SpecTrum Provides / 文件夾提供 | AI Provides / AI 提供 |
|---|---|
| Roles, rules, memory, structure | Reasoning, creativity, judgment |
| 角色、規則、記憶、結構 | 理解、推理、創造力、判斷力 |

**Together** = a complete AI company with 15 specialist roles, persistent memory, and structured workflows.

**合在一起** = 一個完整的 AI 公司，擁有 15 個專業角色、持久記憶和結構化工作流程。

### Secretary 5D Radar / 秘書 5D 雷達

Every message you send is analyzed by the **Secretary** — a hidden routing layer that classifies your intent across 5 dimensions:

每條訊息都會被**秘書**分析 — 一個隱藏的路由層，從 5 個維度分類你的意圖：

| Dimension / 維度 | What It Checks / 檢查內容 |
|---|---|
| **Track** | Task type — writing? research? coding? / 任務類型 |
| **Platform** | Is this a platform-level decision? / 是否需要平台級決策？ |
| **People** | Which role(s) are needed? / 需要哪些角色？ |
| **Style** | Your communication style / 溝通風格 |
| **Supplement** | Background context / 背景上下文 |

**You do NOT need to pick roles manually.** Just talk naturally. The Secretary routes your message to the best role automatically.

**你不需要手動選擇角色。** 自然對話即可。秘書會自動將訊息路由到最佳角色。

But you **can** request specific roles if you want:
- "Switch to T01 Platform Sovereign"
- "以平台主權者身份回答"
- "Let Q06 review this for security risks"

## 5. Memory System / 記憶系統

Q-SpecTrum has two memory files that persist across chat sessions:

Q-SpecTrum 有兩個跨會話持久記憶文件：

| File / 文件 | Purpose / 用途 |
|---|---|
| `MEMORY.md` | Long-term memory — session logs, key decisions, insights, user profile |
| `_HANDOFF/STATUS.md` | Session handoff — current project status snapshot for the next session |

### Saving to Memory / 保存到記憶

Tell the AI to remember important things:

告訴 AI 記住重要的事情：

- "Please save this decision to memory" / "請把這個決策保存到記憶"
- "Remember that we chose React for the frontend" / "記住我們選了 React 做前端"
- "Update my user profile — I'm a backend developer" / "更新用戶畫像 — 我是後端開發者"

The AI will append entries to `MEMORY.md` so future sessions can pick up where you left off.

AI 會將條目追加到 `MEMORY.md`，讓未來的會話可以接續上次的進度。

### Starting a New Session / 開始新會話

Every time you start a new chat session with any AI:

每次與任何 AI 開始新對話時：

1. **Upload the core files** (BOOT.md + SYSTEM-PROMPT.md + ACTION-PROTOCOL.md + KNOWLEDGE-INDEX.md + MEMORY.md + ROLE-REGISTRY.md)
2. **Say**: "Read BOOT.md, check MEMORY.md for context"
3. The AI will activate Q-SpecTrum and restore your previous context

**上傳核心文件**，然後說「讀取 BOOT.md，檢查 MEMORY.md 獲取上下文」，AI 會啟動系統並恢復之前的上下文。

**Tip for Claude Projects / Claude Projects 用戶提示**: If you use Claude Projects, upload all files once to the project knowledge base. Then every new conversation automatically has access — no re-uploading needed.

如果你使用 Claude Projects，只需將文件上傳一次到項目知識庫，之後每次新對話都會自動載入。

## 7. Advanced: Knowledge Base / 進階：知識庫

Q-SpecTrum includes a structured knowledge base in the `AI项目管理/` directory.

Q-SpecTrum 在 `AI项目管理/` 目錄中包含結構化知識庫。

### Database / 數據庫

- **40 tables** in `platform.db` with **85 rows** of core data
- Covers: roles, skills, scenarios, projects, metrics, user growth stages, and more

### Knowledge Resonance / 知識共振

The system uses a resonance formula to retrieve and deposit knowledge:

系統使用共振公式來檢索和存入知識：

```
R = 0.35K + 0.25C + 0.25I - 0.15E
```

| Factor | Meaning / 含義 | Weight |
|---|---|---|
| **K** (Knowledge) | Relevance of stored knowledge / 存儲知識的相關性 | 0.35 |
| **C** (Context) | Current conversation context / 當前對話上下文 | 0.25 |
| **I** (Intent) | User intent signal strength / 用戶意圖信號強度 | 0.25 |
| **E** (Entropy) | Noise and irrelevance penalty / 噪音和無關性懲罰 | -0.15 |

The higher the resonance score, the more relevant the knowledge retrieval.

共振分數越高，知識檢索越相關。

## 8. Advanced: Python Engine Mode / 進階：Python 引擎模式

For users who want a local web interface and REST API:

適用於需要本地 Web 界面和 REST API 的用戶：

### Starting the Engine / 啟動引擎

```bash
python run.py --web    # Web UI + REST API at http://localhost:8765
python run.py          # Interactive CLI chat
python run.py --status # System health check
```

### Web UI / Web 界面

- **Chat interface**: `chat.html` — full-featured chat with role indicators
- **Dashboard**: `dashboard.html` — system overview, role status, metrics
- **Landing page**: `index.html` — project overview and navigation

### REST API

The engine exposes **84 REST endpoints** at `http://localhost:8765`. Key endpoints:

引擎在 `http://localhost:8765` 提供 **84 個 REST 端點**。關鍵端點：

| Endpoint | Method | Purpose / 用途 |
|---|---|---|
| `/api/status` | GET | System health and role status |
| `/api/chat` | POST | Send a message, get routed response |
| `/api/roles` | GET | List all 15 roles and their status |
| `/api/skills` | GET | List available skills |
| `/api/scenarios` | GET | List scenario journeys |
| `/api/memory` | GET | Read current memory state |
| `/api/knowledge/search` | POST | Search the knowledge base |

### LLM Backend Configuration / LLM 後端配置

```bash
QSPECTRUM_LLM=openai OPENAI_API_KEY=sk-xxx python run.py
QSPECTRUM_LLM=anthropic ANTHROPIC_API_KEY=xxx python run.py
QSPECTRUM_LLM=ollama python run.py    # Local models, no API key needed
```

## 9. Tips & Best Practices / 使用技巧

### For Better Results / 獲得更好結果

| Tip / 技巧 | Why / 原因 |
|---|---|
| Be specific in your requests | Better routing = better role match = better answers / 更好的路由 = 更好的角色匹配 |
| Use scenarios for complex projects | Multi-step guidance with automatic role switching / 多步驟引導 + 自動角色切換 |
| Save important decisions to memory | Carry context across sessions / 跨會話保持上下文 |
| Start each new session with "Read BOOT.md" | Re-activates the full system / 重新啟動完整系統 |
| Don't worry about picking the "right" role | The Secretary handles it — just talk naturally / 秘書會處理，自然對話即可 |

### For Power Users / 進階用戶

- **Chain skills and scenarios**: Complete a scenario, then invoke a skill to refine the output. / 完成場景後，調用技能來優化輸出。
- **Use multi-role review**: For critical decisions, ask 2-3 roles to review. / 重要決策可讓 2-3 個角色審查。
- **Customize your profile**: Tell the AI about your background so it adapts responses. / 告訴 AI 你的背景，讓回答更個性化。
- **Build on memory**: Each session adds to your knowledge base — the system gets smarter over time. / 每次會話都會累積知識，系統會越來越懂你。

## 10. FAQ / 常見問題

**Q: Can I use this with ChatGPT? / 可以用 ChatGPT 嗎？**
A: Yes. Q-SpecTrum works with any AI that accepts file uploads — ChatGPT, Claude, Gemini, LLaMA, Ollama, and more. Upload the core files and the AI becomes Q-SpecTrum.

可以。Q-SpecTrum 適用於任何支持文件上傳的 AI。上傳核心文件，AI 就變成 Q-SpecTrum。

---

**Q: Do I need to upload all files every time? / 每次都要上傳所有文件嗎？**
A: For each new chat session, yes — upload the 6 core files. If you use **Claude Projects** or similar features, upload once and every new conversation inherits them automatically.

每次新對話需要上傳 6 個核心文件。如果使用 **Claude Projects** 等功能，只需上傳一次。

---

**Q: How do I update the system? / 如何更新系統？**
A: Replace the files with the new version. Your `MEMORY.md` carries over — it contains your session history, decisions, and user profile. Back it up before updating.

用新版本替換文件即可。`MEMORY.md` 會保留你的會話歷史和決策記錄。更新前請備份。

---

**Q: Can multiple people use the same folder? / 多人可以用同一個文件夾嗎？**
A: Yes. Each person gets their own AI session. The folder is read-only structure — each AI session creates its own conversation context. For shared memory, consider using separate `MEMORY.md` files or a shared one with clear attribution.

可以。每個人有自己的 AI 會話。文件夾是只讀結構。如需共享記憶，可使用各自的 `MEMORY.md` 或帶標註的共享版本。

---

**Q: What if the AI picks the wrong role? / AI 選錯角色怎麼辦？**
A: Just say "Switch to [role name]" and the AI will change. You can also say "That's not quite right — I need a security perspective" and the Secretary will re-route.

直接說「切換到[角色名]」即可。也可以描述你需要的角度，秘書會重新路由。

---

**Q: Do I need Python installed? / 需要安裝 Python 嗎？**
A: No. Python engine mode is **optional** — it adds a web UI and REST API. The core Q-SpecTrum experience is pure chat: upload files to any AI and start talking.

不需要。Python 引擎模式是**可選的**。核心體驗是純對話：上傳文件到任何 AI 即可開始。
